## Arquitectura clave (para entender el código)

- **Namespaces** (en `apps/web/src/i18n/locales/{es-419,en,pt-BR,es-AR}/*.json`): `common, auth,
  charts, account, coach, roster, macros, atleta` (previos) + **`domain`** (5a–5c) + **`legal`** (6).
  Todos preloaded en `apps/web/src/test-setup.ts`.
- **Modelo de locales**: `es-419` = base neutral (tú, **género neutro**) + default global. `es-AR` =
  **overlay voseo** (sólo claves que difieren, subconjunto de es-419). `en` y `pt-BR` = **completos,
  espejo 1:1 de es-419 por clave** (lo fuerza `locales/parity.test.ts`). `en-XA` = pseudo QA (sin catálogo).
- **`domain` ns = capa de PRESENTACIÓN**: el `packages/core` sigue devolviendo datos+ids; el web
  proyecta `id → t('domain:...')`. **No se traduce ningún id/key/family/phaseKey** (contrato persistido).
  Esto mantuvo el generador determinístico de recetas y sus snapshots intactos.
- **Cambio de wire (5c, aditivo)**: `MePlanView.plan.phases[].key` + `plan.currentPhaseKey` +
  `plan.macroId` se agregaron al tipo (`packages/core/src/types/index.ts`), al **schema zod**
  (`packages/core/src/schemas.ts` — antes stripeaba `macroId`) y se pueblan en `buildMePlanView`
  (`packages/core/src/logic/mePlan.ts`). `apps/api` pasa la vista tal cual (sin DTO propio). Habilita
  proyectar las fases del plan al atleta (`domain:macro.{macroId}.phase.{key}.{name,focus}`).
- **Formato locale (5e)**: `apps/web/src/lib/format.ts` → `intlLocale()` mapea **`es-419` → `es-CL`**
  (agrupa miles con punto `1.234.567`; es-419 bajo Intl usa coma). Consumidores usan el hook
  `useFormat()` (number/date/currency).
- **Nombres de movimiento (5d, PR #79)**: **idioma de movimientos es un toggle INDEPENDIENTE** del
  idioma de UI (`apps/web/src/i18n/movementLang.ts`, `useMovementLang.ts`). Antes el `resolved`
  ("es"|"en") NO lo usaba ningún display (sólo búsqueda). Ahora:
  - `apps/web/src/i18n/movementName.ts` → `composeMovementName(id, lang)`: **ES delega en
    `programmableName` del core (byte-idéntico)**; **EN compone** base (`aliasEn`) + modificadores en
    orden inglés ("Hang power snatch (knee)") + 10 complejos EN curados.
  - `useMovementName()` hook → `(id) => string`. ~17 consumidores re-resuelven desde `movementId`.
  - **PT**: `PT_ALIAS_PENDING` en movementName.ts (listo, **NO cableado** — pendiente revisión nativa).

## Convenciones / gotchas (IMPORTANTÍSIMO)

- **CI gates** (`.github/workflows/ci.yml`): el gate que importa es **`typecheck · lint · unit · build`**
  (+ `dependency audit`). El job **`api integration (embedded postgres)` SIEMPRE FALLA** por infra del
  runner (`libicuuc.so.60: cannot open shared object file` en `initdb` de embedded-postgres) — **falla
  igual en `main`; IGNORAR**. `gitleaks` y `GitGuardian` deben estar verdes.
- **Mergeo**: las PRs se crean **draft** → marcar ready (`update_pull_request draft:false`) → **squash merge**.
- **`keySeparator="."`** (i18next default) → claves anidadas son **objetos JSON anidados**, no claves
  con puntos. `nsSeparator=":"` → `t("domain:x")` cruza ns. Los ids con punto (complejos `cx.*`,
  variantes `arranque.potencia...`) se manejan en el composer del web, **no** como claves de catálogo.
- **Verificación local antes de PR**: `pnpm --filter @holy-oly/web exec tsc --noEmit` · `pnpm exec eslint .`
  · `pnpm --filter @holy-oly/web exec vitest run` (594 tests) · `cd apps/web && VITE_API_ENABLED=true pnpm build`.
  Core: `pnpm --filter @holy-oly/core exec vitest run` (471 tests).
- **Trailers de commit**: `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>` +
  `Claude-Session: <url>`. **NUNCA** poner el id de modelo en commits/PRs/código.
